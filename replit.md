# IPL Auction Simulator

A real-time multiplayer IPL auction simulator where friends create rooms, pick franchises, retain players, run live auctions with bidding timers, and get AI-powered squad analysis at the end.

## Run & Operate

- `pnpm --filter @workspace/api-server run dev` — run the API server (port from env)
- `pnpm --filter @workspace/ipl-auction run dev` — run the frontend
- `pnpm run typecheck` — full typecheck across all packages
- `pnpm run typecheck:libs` — rebuild lib declarations (run before leaf typecheck if schema changed)
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API hooks and Zod schemas from the OpenAPI spec
- `pnpm --filter @workspace/db run push` — push DB schema changes (dev only)
- `pnpm --filter @workspace/scripts run seed-ipl` — seed franchises and players data
- Required env: `DATABASE_URL`, `CLERK_SECRET_KEY`, `CLERK_PUBLISHABLE_KEY`, `VITE_CLERK_PUBLISHABLE_KEY`, `GEMINI_API_KEY`

## Stack

- pnpm workspaces, Node.js 24, TypeScript 5.9
- Frontend: React + Vite, Tailwind CSS, Framer Motion, wouter (routing), Clerk auth, Socket.IO client
- API: Express 5 + Socket.IO server
- DB: PostgreSQL + Drizzle ORM
- Auth: Clerk (via `@clerk/express` on backend, `@clerk/react` on frontend)
- AI: Google Gemini (`gemini-2.5-flash`) for squad analysis
- Validation: Zod (`zod/v4`), `drizzle-zod`
- API codegen: Orval (from OpenAPI spec in `lib/api-spec`)
- Build: esbuild (ESM bundle)

## Where things live

- `lib/db/src/schema/` — DB schema (franchises, players, rooms, teams, auction, messages, ai_reports)
- `lib/api-spec/openapi.yaml` — OpenAPI spec (source of truth for all endpoints)
- `lib/api-zod/src/index.ts` — Zod schema barrel (manually curated to avoid TS2308 collisions)
- `lib/api-client-react/src/generated/` — React Query hooks generated by Orval
- `artifacts/api-server/src/routes/` — Express route handlers
- `artifacts/api-server/src/lib/socket.ts` — Socket.IO server setup
- `artifacts/ipl-auction/src/pages/` — React pages (home, dashboard, create, join, room/*)
- `artifacts/ipl-auction/src/hooks/` — Custom hooks (useSocket)
- `scripts/src/seed-ipl.ts` — IPL franchise + player seed data

## Architecture decisions

- Socket.IO path is `/ws/socket.io` (not default) to fit the reverse proxy path routing
- Clerk auth proxy middleware is at `/api/__clerk` — only active in production (no-op in dev)
- `lib/api-zod` uses explicit named exports instead of `export * from "./generated/types"` to avoid TS2308 collisions on shared param type names
- `mergeParams: true` routers need typed `Request<{ code: string }>` generics for TypeScript to know parent params
- AI analysis is generated async (fires after HTTP response) to avoid request timeout

## Product

- **Landing page** — Hero + features + CTA
- **Auth** — Clerk sign-in / sign-up (dark themed, hosted via Replit proxy)
- **Dashboard** — User's rooms, quick create/join
- **Lobby** — Franchise selection, member list, host can start auction
- **Retention phase** — Each team selects players to retain before auction
- **Live auction** — Real-time bidding with countdown timer, bid history, team budgets, chat
- **Results** — AI-powered team rankings, strengths/weaknesses, predicted champion
- **Squad view** — Detailed squad composition per team

## Gotchas

- After changing `lib/db/src/schema`, run `pnpm run typecheck:libs` before typechecking leaf packages
- After running codegen, manually add any new exported types to `lib/api-zod/src/index.ts`
- The API server must be restarted after code changes (it builds on startup via esbuild)
- Socket.IO client must use `{ path: '/ws/socket.io' }` — not the default path
- `nanoid` must stay in `@workspace/api-server` dependencies (used for room code generation)
